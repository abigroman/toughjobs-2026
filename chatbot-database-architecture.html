The Three-Database Intelligence System

One connected system that does three jobs at once: publishes SEO content Google indexes, feeds the chatbot accurate answers, and learns from every conversation — so the bot gets smarter and real customer questions become high-value content. Built to start simple in a no-code tool and scale to a full app.

How The Three Databases Work Together

Think of it as one loop. Conversations reveal what customers ask (DB3). The best answers get approved and stored for the bot (DB2). The public-safe versions get published for Google (DB1). New conversations use those better answers — and the loop tightens every week.

Database 1 · Public

Public Knowledge Base

SEO content Google indexes and customers read. FAQ pages, schema-ready Q&A, URL structure, internal links. Publicly visible.

Database 2 · Private

Private AI Knowledge Base

What the chatbot actually uses. Approved answers, sales scripts, escalation & do-not-say rules, pricing objections. Internal only.

Database 3 · Analytics

Conversation Intelligence

Every conversation logged, scored, and mined. Questions, objections, buying signals, missed answers, content ideas. Internal only.

DB3 surfaces gaps & winners → DB2 stores approved answers → DB1 publishes public-safe content → better answers feed new conversations → back to DB3

Internal Linking Strategy

  • Hub & spoke: each FAQ hub page links down to its individual Q&A entries; each entry links back up to its hub.
  • Cross-link by relevance: a roofing "missed calls" FAQ links to the AI-automations service FAQ.
  • Trade ↔ service: every trade FAQ links to the relevant service pages it references, and vice versa.
  • Local anchoring: location FAQs link to trade + service pages to reinforce local relevance.

Do-Not-Say Examples (global)

  • Never promise "#1 on Google" or guaranteed rankings.
  • Never quote a firm project price without a consultation.
  • Never disparage a named competitor — compare on facts only.
  • Never ask for payment details or SSN.

The Golden Thread

A real question in DB3 → flagged as an insight → approved answer in DB2 → public-safe FAQ in DB1. Every link in that chain is a stored relationship, so you can always trace a published FAQ back to the customer conversations that inspired it.

Automation Ideas

Missed-Answer → Insight

Any message with missed_answer_flag = true auto-creates or increments an insight record. When evidence_count hits a threshold (e.g. 5), notify the admin.

High-Intent Alert

Escalation trigger fires an SMS/Slack to sales with the transcript link, in real time.

Auto Lead Score

On conversation close, run the scoring rules and set lead_score + lead_quality automatically.

Answer Performance

Weekly job counts how often each ai_answer fired and how many led to bookings — surfaces weak answers to rewrite.

Publish Sync

When a public_faq flips to Published, regenerate the page's FAQ schema JSON-LD automatically.

Stale-Content Nudge

Any answer/FAQ not updated in 6 months auto-flags for review.

Core Workflows

Chatbot Improvement Workflow

  1. Bot logs every message; low-confidence answers get missed_answer_flag.
  2. Flags roll up into insights weekly, ranked by evidence_count.
  3. Team drafts a new/updated ai_answer in approved tone.
  4. Admin reviews & approves → answer goes live to the bot.
  5. Answer performance tracked; weak ones cycle back to step 3.

SEO Content Generation Workflow

  1. An approved ai_answer with public value is selected.
  2. Public-safe version drafted as a public_faq (no internal pricing/scripts).
  3. Assign target_keyword, search_intent, URL, and internal links.
  4. Admin approves → status Published, schema auto-generated.
  5. Monitor rankings; refresh entry as intent/keyword data evolves.

Lead Scoring Workflow

  1. During chat, signals are tagged (contact given, pricing asked, urgency, etc.).
  2. On close, additive scoring runs (capped at 100).
  3. Score maps to quality band: High 60–100 / Medium 25–59 / Low 0–24.
  4. High & escalated → real-time sales alert; Medium → nurture; Low → insight only.
  5. Conversion outcome written back to measure score accuracy over time.

Security & Privacy Recommendations

  • Split visibility: DB1 public; DB2 & DB3 locked to authenticated internal users only.
  • Encrypt contact info at rest; restrict who can view the contact_info field.
  • Coarse geo only — city/region, never GPS or full IP lookups.
  • 90-day retention on identifiable records, then anonymize into aggregate insights.
  • No payment/sensitive data ever collected in chat (enforced by do-not-say rules).
  • Role-based access: owner, marketing, sales each see only what they need.
  • Audit log on who approved/published each answer & FAQ.
10

Admin Review Before Public Content Is Published

  1. Draft — content starts at approval_status = Draft. Never visible to bot or public.
  2. Internal check — reviewer confirms accuracy against source_of_truth and tone rules.
  3. Compliance check — run against do_not_say_rules; no guarantees, no firm prices, no competitor bashing.
  4. Approve — owner or marketing lead flips to Approved (bot) or Published (public). Logged with name + date.
  5. Post-publish monitor — track usage/rankings; anything underperforming or stale returns to Draft.

Rule

No answer reaches the chatbot and no page reaches Google without a human approval step. The system drafts; a person decides.

Recommended MVP → Advanced Roadmap

Start in a no-code tool you can run yourself. Graduate to a custom app only when volume justifies it.

MVP — Build First

Tool: Airtable, Notion, or SmartSuite.

  • 3 databases as 3 bases/workspaces, linked by shared services + trades tables.
  • DB2 ai_answers + basic do_not_say + escalation rules.
  • DB1 public_faqs with approval_status & URL fields — export/copy to your site.
  • DB3 manual conversation logging + a simple lead-score formula field.
  • Weekly view: top questions + missed-answer flags.
  • Manual admin approval (a status column + a named approver).

Goal: working loop in ~1–2 weeks, zero code.

Advanced — Build Later

Tool: Supabase / custom web app + bot API.

  • Live bot writes conversations & messages to DB3 automatically.
  • Auto lead scoring, auto insight rollups, real-time escalation alerts.
  • Auto-generated FAQ schema JSON-LD on publish.
  • Answer-performance analytics (fire rate → booking rate).
  • Role-based access, encryption, audit logs, auto-retention purge.
  • Dashboards for owner / sales / content, refreshed live.

Goal: self-improving system that scales across many client businesses.
